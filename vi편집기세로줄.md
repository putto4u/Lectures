## Xshell과 Vim을 활용한 YAML 파일 편집 시 '세로줄' 표시 방법

결론부터 말씀드리면, **Xshell 자체에는 텍스트 에디터 기능이 없으므로 세로줄을 그어주는 기능도 없습니다.** Xshell은 단지 서버의 화면을 보여주는 '모니터' 역할만 하기 때문입니다.

실제로 파일을 편집하는 프로그램인 **`vim` (또는 `vi`) 에디터의 설정을 변경**해야 원하시는 세로줄(인덴트 가이드)이나 현재 커서 위치 표시를 볼 수 있습니다.

YAML 파일 편집 시 들여쓰기(Indent) 실수를 막기 위한 3가지 현실적인 해결책을 알려드립니다.

---

### 1. Vim 설정으로 '현재 위치 세로줄' 만들기 (가장 추천)

리눅스 터미널에서 `vi`나 `vim`으로 파일을 열었을 때, 현재 커서가 위치한 곳에 세로줄(Column Highlight)을 생기게 하여 줄을 맞추기 쉽게 하는 방법입니다.

#### **1.1 일회성 적용 (현재 창에서만)**

`vim`으로 파일을 연 상태에서 다음 명령어를 입력합니다. (`:` 키를 누르고 입력)

```vim
:set cursorcolumn

```

* **효과:** 현재 커서가 있는 세로줄 전체에 색상이 입혀져 위아래 줄 맞춤을 확인하기 매우 편해집니다.

#### **1.2 영구 적용 (`.vimrc` 수정)**

매번 입력하기 귀찮으므로 설정 파일에 저장합니다.

```bash
# 1. 홈 디렉터리의 vim 설정 파일 열기
vi ~/.vimrc

# 2. 파일 맨 아래에 다음 내용 추가 (i 눌러서 입력)
set cursorcolumn    " 현재 커서 위치에 세로줄 표시
set ruler           " 우측 하단에 현재 커서 좌표 표시
set expandtab       " 탭(Tab) 키를 누르면 스페이스바로 변환 (YAML 필수 설정!)
set tabstop=2       " 탭 간격을 2칸으로 설정 (YAML 표준)
set shiftwidth=2    " 자동 들여쓰기 간격을 2칸으로 설정

# 3. 저장하고 나오기 (:wq)

```

---

### 2. 눈에 보이지 않는 공백(Space) 표시하기

YAML은 탭(Tab)을 쓰면 에러가 나고, 스페이스바 2칸 또는 4칸을 엄격하게 지켜야 합니다. 빈 공간에 점이나 기호를 표시해서 칸 수를 세기 좋게 만듭니다.

```vim
" .vimrc 파일에 추가하거나 :set list 입력

set list
set listchars=tab:>-,\ ,trail:·

```

* **효과:** 눈에 안 보이는 공백(Space)이 `·` (가운데 점)으로 표시되어, 들여쓰기가 2칸인지 4칸인지 눈으로 셀 수 있게 됩니다.

---

### 3. Xshell의 '로컬 에디터 연동' 기능 사용 (초보자 추천)

터미널의 검은 화면(Vim)에서 편집하는 것이 너무 어렵다면, Xshell의 강력한 기능인 **"내 컴퓨터에 있는 메모장이나 VS Code로 편집하기"** 기능을 쓰면 됩니다. VS Code에는 완벽한 세로줄 가이드(Indent Rainbow 등)가 있기 때문입니다.

1. **Xshell 화면 상단**에 있는 **[파일 전송]** 아이콘(초록색 폴더 모양)을 클릭하여 **파일 관리자 창**을 엽니다. (또는 `Ctrl+Alt+F`)
2. 편집하고 싶은 YAML 파일을 찾습니다.
3. **마우스 우클릭** > **[다음으로 편집 (Edit with...)]** > **[Notepad]** 또는 **[찾아보기...]**를 눌러 **VS Code**를 등록합니다.
4. 이제 윈도우 창에서 편하게 편집하고 **저장(Ctrl+S)**을 누르면, 자동으로 리눅스 서버에 업로드되어 수정됩니다.
