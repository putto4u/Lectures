## Ubuntu Server 설치 후 숫자 키패드(NumLock) 비활성화 해결

Ubuntu Server를 설치한 직후 키보드 우측의 숫자 키패드가 작동하지 않는 것은 보안이나 하드웨어 결함이 아니라, 리눅스 커널의 기본값에 의해 **NumLock이 꺼져 있기 때문**입니다. 서버 환경은 GUI(데스크탑 화면)가 없으므로 명령어를 통해 부팅 시 자동으로 NumLock이 켜지도록 설정해야 합니다.

---

### 1. 즉각적인 해결 방법

가장 먼저 키보드의 **[NumLock]** 키를 한 번 눌러보세요. 램프에 불이 들어온다면 바로 숫자를 입력할 수 있습니다. 만약 눌러도 반응이 없거나, 매번 부팅할 때마다 누르는 것이 번거롭다면 아래의 자동 설정을 적용해야 합니다.

### 2. 부팅 시 자동 활성화 설정 (numlockx 설치)

Ubuntu Server 환경에서 부팅과 동시에 숫자 키패드를 활성화하려면 `numlockx` 패키지를 사용합니다.

1. **패키지 업데이트 및 설치**
```bash
sudo apt update
sudo apt install numlockx

```


2. **설정 파일 수정**
부팅 시 실행되는 스크립트 파일인 `/etc/rc.local`에 설정을 추가해야 합니다. (최신 Ubuntu 버전에서 해당 파일이 없다면 생성해야 할 수도 있습니다.)
```bash
sudo nano /etc/rc.local

```


3. **내용 입력**
파일 하단(`exit 0`이 있다면 그 윗줄)에 다음 내용을 추가합니다.
```bash
/usr/bin/numlockx on

```



### 3. TTY(터미널) 환경에서의 NumLock 강제 활성화

서버용 터미널(TTY) 자체 설정에서 NumLock을 켜는 방법입니다.

1. **설정 파일 열기**
```bash
sudo nano /etc/console-setup/cached_setup_keyboard.sh

```


*참고: 해당 파일이 없다면 `/etc/default/keyboard` 파일을 확인하여 `KBDREPEATER` 옵션을 점검할 수 있습니다.*
2. **setleds 사용 (고급)**
부팅 스크립트에 아래 명령어를 추가하면 터미널 서비스가 시작될 때 강제로 숫자 키를 활성화합니다.
```bash
# 모든 가상 터미널에 대해 NumLock ON
for tty in /dev/tty[1-6]; do
    setleds -D +num < "$tty"
done

```

### 요약 및 주의사항

* **하드웨어 확인**: 가상 머신(VMware, VirtualBox 등)을 사용 중이라면 호스트 컴퓨터의 NumLock 상태가 게스트 OS로 전달되지 않는 경우가 있습니다. 이 경우 VM 설정의 'Keyboard' 항목을 확인하세요.
* **노트북 키패드**: 일부 노트북은 숫자 키패드와 일반 자판이 겹쳐 있는 경우가 있으니 `Fn` 키 조합을 확인해야 합니다.
