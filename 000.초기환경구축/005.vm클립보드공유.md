VirtualBox 환경에서 Ubuntu Server VM을 설치한 후, 설정에서 '양방향(Bidirectional)'을 활성화했음에도 불구하고 클립보드 공유와 드래그 앤 드롭이 작동하지 않는 문제는 대부분 **Guest Additions** 미설치 또는 **GUI 환경의 부재**로 인해 발생합니다.

---

## 1. Guest Additions 설치 (필수 과정)

VirtualBox의 확장 기능을 사용하려면 게스트 운영체제(Ubuntu) 내부에 전용 드라이버와 유틸리티가 설치되어야 합니다.

### 사전 준비 패키지 설치

터미널에서 아래 명령어를 입력하여 빌드에 필요한 필수 패키지를 먼저 설치합니다.

```bash
sudo apt update
sudo apt install -y build-essential dkms linux-headers-$(uname -r)

```

### Guest Additions ISO 마운트 및 설치

1. VirtualBox 상단 메뉴에서 **[장치] -> [게스트 확장 CD 이미지 삽입...]**을 클릭합니다.
2. 터미널에서 아래 명령어를 순차적으로 실행하여 마운트 및 설치를 진행합니다.

```bash
sudo mount /dev/cdrom /media/cdrom
sudo /media/cdrom/VBoxLinuxAdditions.run

```

3. 설치가 완료되면 시스템을 재부팅합니다.

```bash
sudo reboot

```

## 2. Ubuntu Server의 특성 이해 (GUI 유무)

클립보드 공유와 드래그 앤 드롭은 기본적으로 **X11 또는 Wayland 같은 디스플레이 서버(GUI)** 환경에서 동작하도록 설계되어 있습니다.

* **GUI가 없는 순수 CLI 서버 환경:** 클립보드 공유가 원활하지 않을 수 있습니다. 이 경우 클립보드 대신 **SSH 접속(PuTTY, MobaXterm, VS Code 등)**을 사용하는 것이 표준적인 해결책입니다.
* **Desktop 환경을 설치한 경우:** `gnome-shell`이나 `xfce4` 등이 설치되어 있다면, `VBoxClient` 프로세스가 실행 중인지 확인해야 합니다.

---

## 3. VBoxClient 프로세스 확인 (Desktop 환경 시)

데스크톱 환경을 사용 중인데 작동하지 않는다면, 배경에서 관련 프로세스가 돌고 있는지 확인합니다.

```bash
ps ax | grep VBoxClient

```

만약 `VBoxClient --clipboard`가 보이지 않는다면 수동으로 실행해 봅니다.

```bash
VBoxClient --clipboard
VBoxClient --draganddrop

```

---

## 4. 사용자 그룹 권한 추가

간혹 사용자 계정이 가상머신 장치 제어 그룹에 포함되지 않아 권한 문제로 실패하는 경우가 있습니다. 아래 명령어로 사용자를 `vboxsf` 그룹에 추가합니다.

```bash
sudo usermod -aG vboxsf $USER

```

## 5. 대체 방법: SSH 사용 권장

서버 환경에서는 가상머신 콘솔의 클립보드 기능을 고치는 것보다, 호스트 PC에서 **SSH(Secure Shell)**를 통해 접속하는 것이 훨씬 효율적입니다. SSH를 사용하면 호스트와 게스트 간의 텍스트 복사/붙여넣기가 기본적으로 자유롭습니다.

1. **가상머신 네트워크 설정:** [설정] -> [네트워크] -> [어댑터 1] -> [다음에 연결됨: 어댑터에 브리지] 또는 [NAT - 포트 포워딩(22번 포트)] 설정.
2. **SSH 서버 설치:**
```bash
sudo apt install openssh-server

```


3. **호스트에서 접속:** 터미널이나 커맨드 창에서 `ssh 사용자ID@가상머신IP`로 접속합니다.

## AWS 관련 참고 사항

* **AWS EC2 인스턴스**로 이전하여 운영할 경우, 기본적으로 SSH(Port 22)를 통해 접근하며 클립보드 설정이 필요 없습니다.
* **AWS Cloud9**이나 **Session Manager**를 사용할 경우 브라우저 기반의 클립보드 인터페이스가 기본 제공되나, 인스턴스 사용 시간에 따라 **EC2 요금**이 발생할 수 있으니 주의하시기 바랍니다.
